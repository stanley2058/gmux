title: Concepts
description: Understand Gmux sessions, tabs, panes, clients, servers, and modes.
---

Gmux is a terminal multiplexer. It keeps real terminal processes running and adds a small amount of structure around them.

## Session

A session is a persistent Gmux server namespace. The default `gmux` command attaches to the default session.

Named sessions are separate runtime namespaces:

```bash
gmux new -s work
gmux attach -t work
gmux ls
```

A session owns tabs and panes. Use named sessions when you need separate projects, sockets, and persisted runtime state.

## Tab

A tab is a layout inside a session. Use tabs to separate views like `editor`, `logs`, `server`, or `review`.

Tabs are addressable from the CLI and socket API.

## Pane

A pane is a real terminal. Gmux renders the terminal output, sends input back to the process, and preserves the pane across client detach.

Panes can be split right or down. They can be renamed manually, read from the CLI, sent input, and closed.

## Client and server

By default, Gmux runs as a background server plus one or more attached clients.

The server owns panes and process state. The client is the terminal UI attached to that server.

Detach the client with `ctrl+b d`. The server and panes continue running.

If you want to end the session and stop its panes, stop the server:

```bash
gmux server stop
```

## Modes

Gmux has terminal mode, prefix mode, and optional navigate-style surfaces.

Terminal mode sends keys to the focused pane. Prefix mode waits for one Gmux action after the prefix key.

Press the prefix key, default `ctrl+b`, then an action key such as `c` for a new tab or `%` for a split.

## Mouse UI

Gmux is mouse-native. You can click panes and tabs, drag split borders, select text, and use right-click menus.

If you prefer keyboard-only control, or you want Gmux to stop capturing mouse input, disable mouse capture:

```toml
[ui]
mouse_capture = false
```
