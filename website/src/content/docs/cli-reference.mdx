title: CLI reference
description: Gmux commands for sessions, workspaces, tabs, panes, agents, waits, integrations, and status.
---

Gmux’s CLI talks to the running server over the same local socket API used by integrations and agents.

Most commands print JSON responses. Use them from scripts when you want deterministic automation.

## Launch and status

```bash
gmux                         # launch or attach to the default session
gmux --session work          # launch or attach to a named session
gmux --remote workbox        # attach through SSH, using local keybindings
gmux --remote workbox --remote-keybindings server
gmux --remote workbox --handoff
gmux --no-session            # single-process escape hatch
gmux --default-config        # print default config
gmux update                  # download and install from the configured channel
gmux update --handoff        # opt into live handoff for supported running servers
gmux channel show            # print stable or preview
gmux channel set preview     # opt into preview builds
gmux channel set stable      # return to stable releases
gmux --version               # print version
```

Status commands:

```bash
gmux status
gmux status server
gmux status client
```

## Server

```bash
gmux server
gmux server stop
gmux server reload-config
```

`gmux server` runs the headless server explicitly. Use it for supervised or service-style setups. `reload-config` applies reloadable settings without restarting panes.

## Sessions

```bash
gmux session list [--json]
gmux session attach <name>
gmux session stop <name> [--json]
gmux session delete <name> [--json]
```

Use `default` as the session name when you need to stop the default session explicitly.

## Workspaces

```bash
gmux workspace list
gmux workspace create [--cwd PATH] [--label TEXT] [--focus] [--no-focus]
gmux workspace get <workspace_id>
gmux workspace focus <workspace_id>
gmux workspace rename <workspace_id> <label>
gmux workspace close <workspace_id>
```

Create a workspace without stealing focus:

```bash
gmux workspace create --cwd ~/project --label api --no-focus
```

## Worktrees

```bash
gmux worktree list [--workspace ID | --cwd PATH] [--json]
gmux worktree create [--workspace ID | --cwd PATH] [--branch NAME] [--base REF] [--path PATH] [--label TEXT] [--focus] [--no-focus] [--json]
gmux worktree open [--workspace ID | --cwd PATH] (--path PATH | --branch NAME) [--label TEXT] [--focus] [--no-focus] [--json]
gmux worktree remove --workspace ID [--force] [--json]
```

Worktrees are normal Gmux workspaces with Git checkout provenance. `worktree create` creates a Git worktree checkout, opens it as a workspace, and groups it with the parent repo workspace. Without `--path`, Gmux creates the checkout under `<worktrees.directory>/<repo>/<branch-slug>`.

`workspace close` closes Gmux state only. `worktree remove` is the explicit checkout deletion path; it runs `git worktree remove`, never deletes the branch, and requires `--force` when Git refuses a dirty checkout.

```bash
gmux tab list [--workspace <workspace_id>]

gmux tab get <tab_id>
gmux tab focus <tab_id>
gmux tab rename <tab_id> <label>
gmux tab close <tab_id>
```

## Panes

```bash
gmux pane list [--workspace <workspace_id>]
gmux pane get <pane_id>
gmux pane rename <pane_id> <label>|--clear
gmux pane split <pane_id> --direction right|down [--cwd PATH] [--focus] [--no-focus]
gmux pane close <pane_id>
```

Read output:

```bash
gmux pane read <pane_id> [--source visible|recent|recent-unwrapped] [--lines N]
gmux pane read <pane_id> --source visible --ansi
gmux pane read <pane_id> --source recent-unwrapped --lines 120
```

Send input:

```bash
gmux pane send-text <pane_id> <text>
gmux pane send-keys <pane_id> <key> [key ...]
gmux pane run <pane_id> <command>
```

`pane run` submits text plus Enter atomically. Prefer it over `send-text` plus `send-keys Enter` for commands.

Report agent state from custom hooks:

```bash
gmux pane report-agent <pane_id> \
  --source ID \
  --agent LABEL \
  --state idle|working|blocked|unknown \
  [--message TEXT] \
  [--custom-status TEXT] \
  [--seq N] \
  [--agent-session-id ID] \
  [--agent-session-path PATH]
```

`pane get`, `pane list`, `agent get`, and `agent list` include a read-only `agent_session` object when an official integration has reported a native session reference. If no native session reference is stored, the field is omitted.

Those commands include `foreground_cwd` when Gmux can resolve the cwd of the foreground process controlling the pane. The existing `cwd` field remains the pane/workspace cwd used for labels and follow-cwd behavior.

Report display-only pane metadata without taking over semantic state:

```bash
gmux pane report-metadata <pane_id> \
  --source ID \
  [--agent LABEL] \
  [--applies-to-source ID] \
  [--title TEXT|--clear-title] \
  [--display-agent TEXT|--clear-display-agent] \
  [--custom-status TEXT|--clear-custom-status] \
  [--state-label STATUS=TEXT] \
  [--clear-state-labels] \
  [--seq N] \
  [--ttl-ms N]
```

`STATUS` is one of `idle`, `working`, `blocked`, `done`, or `unknown`. `--agent` is a guard for the authoritative agent label. `--applies-to-source` is a guard for the active lifecycle authority source. Use `--display-agent` to change the visible name.

## Agents

```bash
gmux agent list
gmux agent get <target>
gmux agent read <target> [--source visible|recent|recent-unwrapped] [--lines N] [--format text|ansi] [--ansi]
gmux agent send <target> <text>
gmux agent rename <target> <name>|--clear
gmux agent focus <target>
gmux agent wait <target> --status <idle|working|blocked|unknown> [--timeout MS]
gmux agent attach <target> [--takeover]
gmux agent start <name> [--cwd PATH] [--workspace ID] [--tab ID] [--split right|down] [--focus|--no-focus] -- <argv...>
```

Agent targets can be terminal IDs, unique agent names, detected or reported agent labels, or legacy pane IDs. Names and labels are agent identities. Terminal IDs and legacy pane IDs are low-level escape hatches.

`agent read` reads the resolved terminal stream. `agent send` writes literal text to that stream. `agent get`, `agent focus`, `agent wait`, and `agent attach` require the resolved terminal to have agent identity. `agent rename` can assign that identity.

Use `pane send-text`, `pane send-keys`, `pane run`, and `terminal attach` for ordinary terminals, servers, tests, shells, or low-level terminal control. Use `pane run` when you want to submit a command with Enter.

## Direct terminal attach

```bash
gmux terminal attach <terminal_id> [--takeover]
```

Detach from direct attach with `ctrl+b q`. Send literal `ctrl+b` with `ctrl+b ctrl+b`.

## Waits

Wait for output in a pane:

```bash
gmux wait output <pane_id> --match <text> [--source visible|recent|recent-unwrapped] [--lines N] [--timeout MS] [--regex] [--raw]
```

Wait for pane agent state:

```bash
gmux wait agent-status <pane_id> --status <idle|working|blocked|done|unknown> [--timeout MS]
```

Use `wait output` for normal commands and servers. Use `wait agent-status` for coding agents.

## Read sources

| Source | Meaning |
| --- | --- |
| `visible` | Current rendered screen. Best for UI feedback loops. |
| `recent` | Recent scrollback with terminal wrapping. |
| `recent-unwrapped` | Recent scrollback without soft wrapping. Best for logs. |

## Environment variables

| Variable | Purpose |
| --- | --- |
| `GMUX_CONFIG_PATH` | Override the config file path. |
| `GMUX_SESSION` | Select a named session for CLI commands. |
| `GMUX_SOCKET_PATH` | Low-level socket path override. |
| `GMUX_LOG` | Set log filter, for example `GMUX_LOG=gmux=debug`. |
| `GMUX_DISABLE_SOUND` | Disable sound playback even when sound notifications are enabled. |
