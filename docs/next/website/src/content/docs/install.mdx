---
title: Install Gmux
description: Install, update, and verify Gmux on Linux and macOS.
---

Gmux ships as a single binary for Linux and macOS.

## Install

Run the installer:

```bash
curl -fsSL https://gmux.dev/install.sh | sh
```

The installer downloads the right release binary for your platform and places it on your PATH.

## Install with Homebrew

If you already use Homebrew:

```bash
brew install gmux
```

## Install with mise

If you already use mise:

```bash
mise use -g gmux
```

## Verify

Start Gmux:

```bash
gmux
```

If your shell cannot find `gmux`, restart the terminal or check that the install directory is on your PATH.

## Update

Gmux checks for new releases and notifies you in the app. You can update manually:

```bash
gmux update
```

`gmux update` is for installs managed by Gmux's own installer. Homebrew, mise, and Nix installs are updated through those package managers instead.

Gmux uses the stable update channel by default. To opt into preview builds from `master`, set the channel:

```bash
gmux channel set preview
```

Switch back to stable the same way:

```bash
gmux channel set stable
```

For direct installs, changing channels also checks that channel and installs its latest binary. If that update fails, run `gmux update` to retry from the configured channel.

Preview builds are GitHub prereleases published from the current development branch. They are useful when you want fixes before the next stable release, but they can regress. Homebrew, mise, and Nix installs do not use the preview channel.

By default, `gmux update` installs the new binary and leaves compatible running sessions alone. If an update changes Gmux's client/server protocol, Gmux asks whether to stop the old server after installing. Stop the old server to use the new version. Stopping exits pane processes. For the default session, run `gmux server stop`, then run `gmux` again. For a named session, run `gmux session stop <name>`, then run `gmux session attach <name>` again.

To opt into experimental live server handoff for supported running sessions, run:

```bash
gmux update --handoff
```

Live handoff does not apply to Homebrew, mise, or Nix package-manager updates. For those installs, update with the package manager, then restart that Gmux session when you are ready to use the new server. If a running session still uses the old server, stop it with `gmux server stop` or `gmux session stop <name>`, then run Gmux again.

## Install with Nix

If you already use Nix, Gmux provides a flake that builds Gmux from source:

```bash
nix run github:ogulcancelik/gmux/v0.x.y
nix build github:ogulcancelik/gmux/v0.x.y
nix profile install github:ogulcancelik/gmux/v0.x.y
```

Replace `v0.x.y` with the latest release tag. You can omit the tag to track `master`, but release tags are recommended for normal installs.

The flake also exposes a development shell:

```bash
nix develop github:ogulcancelik/gmux
```

Update through the same Nix workflow you used to install Gmux. For a profile install, list your profile entries and upgrade the Gmux entry:

```bash
nix profile list
nix profile upgrade <index-or-name>
```

If Gmux is an input in your own flake, update that input and rebuild your system, Home Manager, or development environment:

```bash
nix flake update gmux
```

## Download manually

You can also download a binary from [GitHub releases](https://github.com/ogulcancelik/gmux/releases).

Choose the asset that matches your system:

| System | Asset |
| --- | --- |
| Linux x86_64 | `gmux-linux-x86_64` |
| Linux aarch64 | `gmux-linux-aarch64` |
| macOS Intel | `gmux-macos-x86_64` |
| macOS Apple silicon | `gmux-macos-aarch64` |

Make it executable and move it somewhere on your PATH.

```bash
chmod +x gmux-linux-x86_64
mv gmux-linux-x86_64 ~/.local/bin/gmux
```

## Requirements

Gmux supports Linux and macOS. Native Windows support is not available yet; use Gmux inside WSL for now.
