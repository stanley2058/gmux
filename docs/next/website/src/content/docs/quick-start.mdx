---
title: Quick start
description: Start a Gmux session, split panes, create tabs, and detach safely.
---

Start Gmux from any project directory:

```bash
gmux
```

Gmux launches or attaches to your default background session. You do not manage sockets. If you detach, panes keep running.

For the local, SSH, and `gmux --remote` workflows, see [How to work with Gmux](/docs/how-to-work/).

## Work in a session

The first screen is a shell in your session's first tab. Use it like a normal terminal, then split panes or add tabs as your work grows.

For separate projects or machines, use named sessions:

```bash
gmux new -s work
gmux attach -t work
gmux ls
```

Each named session has its own tabs, panes, socket, and persisted runtime state.

## Keyboard control

Press `ctrl+b` to enter prefix mode, then press an action key.

Common actions:

| Action | Key |
| --- | --- |
| Split right | `prefix+v` |
| Split down | `prefix+minus` |
| New tab | `prefix+c` |
| Next / previous tab | `prefix+n` / `prefix+p` |
| Select tab | `prefix+1..9` |
| Detach client | `prefix+d` |

After detaching, run `gmux` again to reattach to the same session.

## Use the mouse

Gmux is mouse-native. You can click panes and tabs, drag borders, drag-select text to copy it to your clipboard, double-click a token to copy it directly, and use right-click menus. Copying does not require Ctrl+C.

If you configure `ui.right_click_passthrough_modifier`, that modifier plus right-click sends right-click, hold, and drag gestures to mouse-reporting pane apps.

Ctrl-click opens pane links when your terminal sends the modified click to Gmux. This works for OSC 8 hyperlinks and visible `http://` or `https://` URLs. The portable terminal-native fallback is Shift-Ctrl-click on Linux or Shift-Cmd-click on macOS.

## Copy from the keyboard

Press `prefix+[` to enter copy mode for the focused pane. Use `h/j/k/l`, `w/b/e`, and `{`/`}` to move, `v` or Space to start a selection, `y` or Enter to copy it, and `q` or Esc to leave without copying.

## Manage named sessions

Named sessions are separate Gmux server namespaces. Use them when you want fully separate runtime state.

```bash
gmux session list
gmux session attach work
gmux session attach side-project
gmux session stop work
gmux session delete side-project
```

Use named sessions when you need isolation between sets of panes, sockets, and persistent state.

## Attach from another machine

Run Gmux where the work lives. If the code and credentials are on a server, either SSH there and run Gmux like a terminal multiplexer:

```bash
ssh you@server
gmux
```

Or attach from your local terminal through SSH:

```bash
gmux --remote workbox
gmux --remote ssh://you@server:2222
```

For repeat targets, put the host in your SSH config. See [How to work with Gmux](/docs/how-to-work/) for the full model.
